# KMBA菁英計畫 2026 — 智慧客服

KT&G 大韓菸草 **KMBA菁英計畫** 官方 FAQ 智慧客服（靜態版）。  
店家可輸入問題，即時取得活動任務、抽獎券、超商禮券、區域競賽等說明。

**線上版本：** [https://zxcvaden-hub.github.io/KMBA-FAQ/](https://zxcvaden-hub.github.io/KMBA-FAQ/)

---

## 功能特色

- **智能語意匹配**：支援錯字、少打字、同義說法（非單純關鍵字比對）
- **雙語回覆**：中文提問 → 繁體中文；英文提問 → English
- **超商禮券／抽獎券分離**：避免「禮券」被誤判為「抽獎券」
- **快捷按鈕**：日常任務說明、抽獎規則、超商禮券說明、拜訪任務說明
- **安全護欄**：菸品售價、法規等問題引導聯繫區域業務
- **純靜態部署**：GitHub Pages 即可運作，無需後端伺服器

1. 將以下檔案 push 至 `main` 分支：
   - `index.html`
   - `kb-engine.js`
   - `kmba-logo.png`
2. Repo → **Settings** → **Pages**
3. Source 選 **Deploy from a branch** → Branch: `main` → Folder: `/ (root)`
4. 等待 1～5 分鐘後重新整理頁面

## 幸運星抽獎工具（`lucky/`）

獨立於 AI 客服的**公開、可驗證**抽獎工具，與客服共用同一個 GitHub Pages 網站，放在 `lucky/` 子資料夾，純前端、離線運作、無需後端。

### 檔案

| 檔案 | 用途 |
|------|------|
| `lucky/index.html` | 抽獎介面 |
| `lucky/lucky.js` | 名單解析、SHA-256 快照、可驗證加權抽選、動畫 |
| `lucky/banner.png` | 頁首主視覺 |
| `lucky/result.json` | 開獎後由主辦放入，供觀眾頁跨裝置讀取（選用） |

### 三個入口

| 模式 | 網址 / 通關語 | 說明 |
|------|------|------|
| 主辦設定 | `.../lucky/` | 完整功能：上傳名單、設定種子、開獎 |
| 觀眾觀看 | `.../lucky/?view=1` | 只能播放動畫與看結果，無法上傳 |
| 客服後門（觀眾） | 在 AI 客服輸入 `luckydrawlist` | 跳轉到觀眾模式（只能看） |
| 客服後門（設定） | 在 AI 客服輸入 `luckydrawsetting` | 跳轉到主辦設定頁（完整功能） |

> 後門通關語定義於根目錄 `index.html`（不分大小寫）。觀眾模式讀取結果順序：先抓 `lucky/result.json`，無則退回讀本機暫存。

### 抽獎規則

- **得獎人數：** 預設 15 位（可調整），獎項相同：價值新台幣 $5,000 元超商禮券。
- **加權抽選：** 每張抽獎券為一個權重，券越多中獎機率越高；同一參加者最多中獎一次（不放回，採 Efraimidis–Spirakis 演算法）。
- **重複參加者：** 預設「同姓名合併、券數加總」，可切換為「同姓名＋店別」或「不合併」，並可勾選忽略英文大小寫／空白差異。

### 公平公正公開的機制

1. **名單快照（SHA-256）：** 開獎前先公告整份名單的雜湊值；名單被更動則快照碼必變，無法事後造假。
2. **可驗證隨機種子：** 採用「指定某期台灣彩券開獎號碼」當種子——公開、不可預測、無人能操控。
3. **確定性演算法：** 任何人以「相同名單 + 相同種子」於本工具重跑，會得到**完全相同**的得獎名單。

### 開獎 SOP（給主辦方）

1. **開獎前公告：** 上傳 CSV 名單 → 解析產生「名單快照碼」→ 對外公告此碼，並聲明將採用哪一期彩券號碼當種子。建議先按「下載解析後名單」存證。
2. **等彩券開獎：** 待指定期彩券號碼開出（此時名單已鎖定、種子由彩券決定，兩邊都無法再動手腳）。
3. **執行開獎：** 將彩券號碼填入「公開種子」→ 一鍵開獎 → 顯示得獎名單與動畫。
4. **公布結果：** 用「複製公告文字」或「下載結果（CSV）」對外公告（內含快照碼、種子、得獎名單）。
5. **（跨裝置觀眾頁，選用）** 點「下載觀眾頁結果檔（result.json）」→ 將 `result.json` 放入 `lucky/` 並 push 到 GitHub → 任何人輸入通關語即可在自己的裝置看到同一份結果。
6. **接受驗證：** 有疑慮者可自行以相同名單＋種子重跑，結果一致即證明公正。

### CSV 名單格式

含標題列即可，系統自動辨識欄位（姓名／店別／區域／抽獎券總數），支援 UTF-8 與 Big5（Excel 另存）編碼。也可改用手動貼上（每行 `姓名,抽獎券數`）。
